# 🛡️ [게임 이름: Last War Survival] 
> **유니티 게임 엔진 구조 및 상호작용 이해를 위한 실습 프로젝트**

## 🎯 프로젝트 소개 (Project Overview)
이 프로젝트는 게임 QA 직무를 준비하며, **유니티 엔진의 기본 작동 원리와 오브젝트 간의 상호작용 구조**를 이해하기 위해 제작한 '캐주얼 슈팅 디펜스' 게임입니다.
단순한 게임 구현을 넘어, 인게임 요소(플레이어, 적, 장애물, UI)들이 내부적으로 어떻게 데이터를 주고받으며, **어떤 로직에서 버그나 최적화 문제가 발생할 수 있는지 파악하는 시야**를 기르는 데 중점을 두었습니다.

## 🔍 QA 관점에서의 핵심 학습 내용

이 프로젝트를 직접 개발하며 다음과 같은 엔진의 동작 원리와 검증 포인트를 학습했습니다.

**1. 오브젝트 충돌 및 상호작용의 이해 (`Collider` & `Trigger`)**
* `OnTriggerEnter`를 활용하여 총알, 적, 장애물(오크통, 숫자 벽) 간의 충돌 이벤트를 직접 구현했습니다.
* **[QA Point]** 게임 내 피격 판정 누락이나 중복 충돌 버그가 발생하는 원리(레이어 설정 오류, 물리 엔진 프레임 차이 등)를 구조적으로 이해하게 되었습니다.

**2. 객체 생명주기와 메모리 최적화 체감 (`Instantiate` & `Destroy`)**
* 시간 경과(Wave)에 따라 적과 장애물을 무작위로 동적 생성(`Instantiate`)하고, 화면 밖(DeadLine)으로 벗어난 객체는 즉시 파괴(`Destroy`)하는 로직을 적용했습니다.
* **[QA Point]** 파괴되지 않은 가비지(Garbage) 객체가 씬에 누적될 때 발생하는 프레임 드랍 현상을 직접 확인하며, 리소스 관리와 최적화 검증의 중요성을 체감했습니다.

**3. 상태 관리(FSM)와 예외 상황(Edge Case) 파악 (`Enum`)**
* `Enum`을 사용해 캐릭터의 생존/사망 상태(`SoldierState`)와 게임의 진행 상태(`PlayerState`)를 분리하여 설계했습니다.
* **[QA Point]** 캐릭터의 상태가 전환될 때 겹치거나 꼬일 수 있는 여러 예외 상황(Edge Case)을 시뮬레이션하고 예측하는 훈련이 되었습니다.

## 📂 주요 스크립트 구조 (Scripts Overview)
* `GameManager` / `SpawnManager`: 씬 전체의 상태 통제 및 동적 오브젝트 소환, 레벨 디자인 제어
* `Player` / `Soldier` / `Enemy`: 개별 유닛의 상태(Enum) 관리 및 체력 증감 로직
* `Bullet` / `Wall` / `WeaponBarrel`: 태그(Tag) 식별을 통한 조건부 충돌 상호작용 처리
* `HUD`: 3D 월드 좌표계를 2D 스크린 좌표계로 변환하여 UI를 매핑하는 로직 제어
